# AI 문서화 플로우 (StartHub)

> ⚠️ **주의**: 이 문서는 **참고용**입니다. 실제 구현 시 세부 사항은 변경될 수 있으며, 최신 스펙은 별도 확인이 필요합니다.

> **목적**: 사용자가 지원 사업 문서를 AI 도움으로 작성하는 전체 흐름을 정의합니다.  
> **핵심 분기**: 지원 사업 양식 파일(템플릿) 존재 여부에 따라 두 갈래로 나뉩니다.

---

## 전체 플로우 개요

```
[문서 목록 화면]
   │
   ▼  사용자가 + 버튼 클릭
[템플릿 업로드 화면]
   │
   ▼
[양식 파일 존재 여부 판단]
   │
   ├── YES (파일 업로드 완료)
   │      ▼
   │   [양식 기반 AI 질문 생성] ──▶ [문서 작성] ──▶ [수정 (드래그 & 직접 편집)] ──▶ [저장] ──▶ [문서 목록]
   │
   └── NO  (건너뛰기 클릭)
          ▼
       [임의 질문] ─────▶ [문서 생성] ──▶ [수정 (드래그 & 직접 편집)] ──▶ [저장] ──▶ [문서 목록]
```

---

## Step 0. 문서 목록 화면 — 진입점

**화면 설명**  
AI 문서화. 사용자가 보유한 문서 목록이 카드 그리드로 표시됩니다. 새 문서를 만들려면 `+` 버튼을 클릭합니다.

**UI 구성요소**
- 좌측 상단 `+` 카드 버튼: **새 문서 생성 진입점**
- 문서 카드 그리드
  - 썸네일 (문서 미리보기)
  - 문서명 (예: `docs2`, `스타트허브 BMC`)
  - 작성일 (예: `2025.04.09`)
- 기존 문서 카드 클릭 → Step C-1 편집 화면으로 이동

**분기 조건**
- `+` 버튼 클릭 → [Step A-1 템플릿 업로드 화면]으로 이동

---

## 경로 A — 양식 파일 있음

### Step A-1. 템플릿 업로드 화면

**화면 설명**  
사용자가 지원 사업 공고의 공식 양식 파일을 업로드합니다.

**UI 구성요소**
- 헤더 타이틀: `"템플릿을 업로드하세요."`
- 서브 텍스트: `"지원 사업 계획서 양식 파일을 업로드 하면 피팅을 기반으로 질문을 생성합니다."`
- 파일 드래그앤드롭 영역 (점선 박스)
  - 지원 형식: PDF, DOCX, HWP
  - 최대 용량: 20MB
  - `파일 선택하기` 버튼
- 업로드된 파일 목록 (파일명 + 용량 + 삭제 버튼)
- 하단 액션 버튼: `[다음 단계]` `[건너뛰기]`

**AI가 이 화면에서 받는 입력값**
```
- file: 업로드된 양식 파일 (PDF / DOCX / HWP)
- 파일 파싱 결과: 섹션 제목, 항목명, 입력 필드 구조
```

**분기 조건**
- 파일 업로드 완료 → [Step A-2]로 이동
- `건너뛰기` 클릭 → 경로 B [Step B-1]로 이동

---

### Step A-2. 양식 기반 AI 질문 생성 화면

**화면 설명**  
업로드된 양식의 항목을 분석하여 사용자에게 맞춤 질문을 제시합니다.

**UI 구성요소**
- 질문 목록 (번호 + 질문 텍스트)
  - 예: `해당 문서는 누구(타깃 사용자)에게 발생하나요?`
  - 예: `문서 전체에 걸쳐 유지해야 할 톤앤매너(Tone & Manner)는 어떠해야 하나요?`
    - 선택지 칩 UI: `전문적이고 신뢰감 있는` / `감성적 & 실무적인` / `바로 설명`
  - 예: `사용자가 겪는 가장 큰 불편은 무엇인가요?`
    - 텍스트 입력 필드
- 하단 CTA 버튼: `문서 생성하기`

**AI가 이 화면에서 받는 입력값**
```
- 양식 파일의 섹션 구조
- 사용자 답변 (타깃, 톤앤매너, 핵심 불편 등)
→ 이 값을 조합해 문서 초안 생성 프롬프트를 구성
```

---

## 경로 B — 양식 파일 없음

### Step B-1. 임의 질문 화면

**화면 설명**  
양식 없이 일반적인 지원 사업 문서 구조 기반으로 필수 질문이 나타난다.

**UI 구성요소**
- Step A-2와 동일한 질문 목록 UI
- 단, 질문 내용은 범용 템플릿 기반 (양식 파일 비의존)
- 하단 CTA 버튼: `문서 생성하기`

**AI가 이 화면에서 받는 입력값**
```
- 사용자 답변만 (양식 구조 없음)
→ 범용 사업계획서 구조로 문서 초안 생성
```

---

## 공통 경로 — 문서 작성 / 수정 / 저장

### Step C-1. 문서 편집 화면 (작성 결과)

**화면 설명**  
AI가 생성한 문서 초안을 사용자가 직접 편집하는 리치 텍스트 에디터 화면입니다.

**UI 구성요소**
- 상단 툴바: Bold / Italic / Underline / 리스트
- 저장 상태 표시: `실시간 자동 저장 중`
- 우측 사이드바 — **문서 정보 패널**
  - 의뢰자 수
  - 공자 수 (1,284자)
  - 작업 히스토리 (이전 버전 목록)
  - 버튼: `[내보내기]` `[저장]`
- 본문 영역: AI 생성 텍스트 (편집 가능)
  - 일부 텍스트에 AI 제안 하이라이트 + 말풍선 툴팁 표시

**수정 인터랙션**
- **드래그로 섹션 순서 변경**: 각 섹션(단락/항목) 좌측의 드래그 핸들(`⠿`)을 잡고 드래그하여 순서를 재배치
  - 드래그 가능한 단위: 섹션 블록 (제목 + 본문 묶음)
  - 드래그 중 다른 섹션 위로 올라가면 삽입 위치 표시선 노출
  - 드롭 후 순서 즉시 반영 + 자동 저장
- 사용자가 본문 직접 텍스트 편집 가능
- AI 하이라이트 클릭 → 대안 문구 제안 툴팁 노출
- 자동 저장: 편집 및 드래그 후 실시간 반영

---

### Step C-2. 저장 및 문서 목록 복귀

**화면 설명**  
저장된 문서가 My 비즈니스 페이지의 문서 그리드에 추가됩니다. 플로우 종료 후 Step 0으로 복귀합니다.

**UI 구성요소**
- 상단 `+` 카드 버튼: 다음 신규 문서 생성의 진입점 (Step 0 → Step A-1 반복)
- 방금 저장된 문서가 그리드 상단에 추가됨
- 문서 카드 클릭 → Step C-1 편집 화면으로 재진입 가능

## 상태 흐름 요약

| 단계 | 경로 A (양식 있음) | 경로 B (양식 없음) |
|------|-------------------|-------------------|
| **0** | 문서 목록에서 `+` 클릭 | 문서 목록에서 `+` 클릭 |
| **1** | 템플릿 파일 업로드 | 템플릿 업로드 화면에서 `건너뛰기` |
| **2** | 양식 기반 질문 생성 | 범용 질문 생성 |
| **3** | 문서 초안 작성 | 문서 초안 생성 |
| **4** | 에디터에서 수정 (드래그 재배치 + 텍스트 편집) | 에디터에서 수정 (드래그 재배치 + 텍스트 편집) |
| **5** | 저장 → 문서 목록 복귀 | 저장 → 문서 목록 복귀 |

---

## AI 컨텍스트 요약 (프롬프트 설계용)

AI가 각 단계에서 사용하는 핵심 입력/출력은 다음과 같습니다.

```
[진입 — Step 0]
Trigger: 사용자가 문서 목록 화면의 + 버튼 클릭
→ 항상 템플릿 업로드 화면(Step A-1)으로 이동

[경로 A]
Input:  양식 파일(구조) + 사용자 질문 답변
Output: 양식 섹션에 맞춰 채워진 문서 초안

[경로 B]
Input:  사용자 질문 답변만 (양식 구조 없음)
Output: 범용 사업계획서 구조의 문서 초안

[공통 편집 단계 — Step C-1]
Input:  사용자 드래그(섹션 순서 변경) + 텍스트 직접 편집 + AI 하이라이트 피드백
Output: 최종 저장 문서
```
